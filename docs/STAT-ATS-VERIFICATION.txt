Documents all ATS verification work performed via CI probes.

and any corrections made to SEED_COMPANIES as a result.

═══════════════════════════════════════════════════════════════
GENERAL METHODOLOGY
═══════════════════════════════════════════════════════════════

For each health system:
  1. Fetch official careers page (HTML GET with browser UA)
  2. Scan HTML for ATS fingerprints (Workday, Avature, Taleo, etc.)
  3. Probe ATS-specific endpoints directly:
     Workday: POST /wday/cxs/{tenant}/{slug}/jobs (JSON API)
     BrassRing: GET sjobs.brassring.com/?partnerid={id}&siteid={sid}
     Avature: GET avature.net/careers (HTML GET)
     Phenom: GET cdn.phenompeople.com fingerprint in HTML
  4. Record HTTP status, ATS signals, redirect final URLs
  5. Update SEED_COMPANIES and document finding here

Probe scripts: scripts/analyze_hc.py, scripts/analyze_job_page.py
Probe workflows: .github/workflows/umiami-*.yml, workday-ucsf-verify.yml

WORKDAY PATH SLUG FIX — VERIFIED 2026-06-06
═══════════════════════════════════════════════════════════════

BUG: fetchWorkday() used hardcoded slug 'External_Career_Site' for all tenants.
  This is wrong — every Workday tenant has a unique slug.
  All Workday API calls were returning 422 for the wrong slug.

FIX (commit 86f17bc):
  Slug is now derived from the last path segment of company.url.
  Example: url = 'jhhs.wd5.myworkdayjobs.com/en-US/JHH_External_Positions'
    tenant = 'jhhs'
    slug   = 'JHH_External_Positions'
    apiUrl = '.../wday/cxs/jhhs/JHH_External_Positions/jobs'

VERIFICATION STATUS:
  The fix is logically correct but requires that company.url in
  SEED_COMPANIES has the correct path slug.
  Not all 181 companies have been individually verified.
  Recommended: TYPE D audit — probe each Workday URL via CI.
  Expected failure rate: unknown. Some health systems use custom
  career domains that don't follow the myworkdayjobs.com pattern.

PROBE INFRASTRUCTURE REFERENCE
═══════════════════════════════════════════════════════════════

GitHub Actions workflow dispatch pattern:
  1. Commit workflow file to .github/workflows/
  2. Wait 90s for GitHub trigger indexing (new files need this)
  3. POST dispatch: api.github.com/repos/.../workflows/{id}/dispatches
  4. Wait 60-90s for run completion
  5. Pull outbox/*.txt via git pull

YAML heredoc gotcha:
  Python heredocs (<< 'PYEOF') inside YAML run: blocks cause GitHub
  to invalidate the workflow_dispatch trigger without warning.
  Fix: commit Python as separate files in scripts/, call from workflow.

Workflow ID discovery:
  GET api.github.com/repos/.../actions/workflows
  Returns id, name, path, state for each workflow.
  Use ID (integer) for dispatch when filename hasn't been indexed yet.

Reusing indexed workflows:
  The fastest approach is to repurpose an already-indexed workflow
  by updating its content (the trigger stays registered).
  290576888 (hc-verify.yml) is the primary reusable probe vehicle.
