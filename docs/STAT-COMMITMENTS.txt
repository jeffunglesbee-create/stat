RULE: HOW TO USE THIS FILE
═══════════════════════════════════════════════════════════════

Before starting a build session:
  Read this file. If an open item is in scope, build it.
  Do not build new features while open items remain unaddressed
  unless they are explicitly deferred with a documented reason.

When shipping a promised feature:
  Move it from OPEN to VERIFIED BUILT.
  Add WHERE and VERIFIED lines.
  Commit this file with the code.

When making a new promise in a session:
  Add it to OPEN before writing any code.
  This creates accountability before the promise is made,
  not after the session ends.
