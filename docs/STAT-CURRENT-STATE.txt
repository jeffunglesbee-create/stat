WHAT STAT IS
═══════════════════════════════════════════════════════════════

STAT = "Straight To A Target"
A personal job intelligence system for Epic / Healthcare IT roles,
built on Cloudflare Workers Plus.

STAT is completely separate from FIELD (sports intelligence PWA).
No shared infrastructure. No shared design tokens. No shared logic.

PURPOSE:
Monitor 181 healthcare employers directly at the ATS source every
30-120 seconds, apply keyword matching + AI-powered resume scoring,
infer missing salaries from government and peer sources, and alert
the moment a matching Epic/EHR/Healthcare IT role appears.
Also monitors via HiringCafe wide-net scrape (1-minute cron) to
catch the long tail of employers not in the seed list.

DESIGN PRINCIPLE: DO NOT ASSUME
Every displayed salary traces to a verifiable source.
Every alert traces to a job confirmed live at the ATS source.
Nothing is fabricated or guessed.

DURABLE OBJECTS — THREE CLASSES
═══════════════════════════════════════════════════════════════

CLASS 1: PlatformDO (binding: PLATFORM_DO)
  One instance per ATS platform (Greenhouse, Lever, Ashby, Workday,
  iCIMS, SuccessFactors, Taleo). Round-robin polls all companies
  on that ATS. One alarm per platform.
  Alarm intervals: greenhouse/lever/ashby 120s, workday/icims 240s,
                   successfactors/taleo 480s, Saturday 20min.

CLASS 2: BatchPollerDO (binding: BATCH_POLLER_DO)
  ONE globally (named 'batch-poller'). Covers BATCH_WATCHLIST —
  general employers, logistics, CS companies. Round-robin every 4min.
  50 companies per alarm cycle, 400ms between fetches.

CLASS 3: StateStoreDO (binding: STATE_STORE)
  ONE globally (named 'stat-state'). Shared state store.
  Keys: recent_matches (200), seen_ids (5000 cap), company_list,
        profile (candidate JSON), unmatched_jobs (500 cap, Browse tab).

ALERT ROUTING
═══════════════════════════════════════════════════════════════

P1 (Epic / EHR / Healthcare IT):
  Pushover HIGH priority — bypasses DND, siren sound. Per-job.

P2 (Data / SQL / Analytics):
  Pushover normal priority — batched into one push.

P3 (Product / Project / IT):
  Email only — no push.

ALL: email digest with fit score, salary, cover letter opener.
Fit scoring (Gemini) can downgrade P1 to email-only if score < threshold.
Requires stored candidate profile (POST /extract-profile first).

HIRINGCAFE — CURRENT STATUS (2026-06-07)
═══════════════════════════════════════════════════════════════

Cron: 1-minute, runHiringCafeScrape()
Calls per tick: 2 (one per env: remote, hybrid)
  Previously 62 (31 terms x 2) — collapsed 2026-06-07.
  All calls return same 152-item global feed regardless of keyword.
  Keyword filtering happens in matchJob() downstream.

Feed key: __NEXT_DATA__.props.pageProps.ssrHits
  FIXED 2026-06-07: was pp.jobs (broken for unknown duration).

Backend: Internal Elasticsearch cluster (confirmed via ssrTimings.esLatencyMs).
  Not Algolia. No public API. Credentials never exposed to client.

Description second-pass: enrich.js fetches hiring.cafe/job/{hc.requisitionId}
  Returns NEXT_DATA.props.pageProps.job.job_information.description (full HTML).
  Fires only after matchJob() hit — minimal HTTP cost.
  See STAT-HIRINGCAFE.txt for full investigation.

CLAUDE INLINE REVIEW (2026-06-07)
═══════════════════════════════════════════════════════════════

Each match card in the Matches tab has a "✦ Review" button.
On click: streams Claude analysis inline, no tab switch, no page leave.

Worker: POST /review (SSE streaming)
  Model: claude-haiku-4-5-20251001
  Format: VERDICT / TOP SIGNALS / DEALBREAKER / QUICK TAKE
  Context: job fields + stored candidate profile (optional)
  Requires ANTHROPIC_API_KEY secret.

UI: toggleReview(jobId) in ui.html
  Token-by-token SSE render. reviewCache for instant repeat.
  See STAT-CLAUDE-REVIEW.txt for full spec.
