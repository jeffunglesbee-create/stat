WHAT STAT IS
═══════════════════════════════════════════════════════════════

STAT = "Straight To A Target"
A personal job intelligence system for Epic / Healthcare IT roles,
built on Cloudflare Workers Plus.

STAT is completely separate from FIELD (sports intelligence PWA).
No shared infrastructure. No shared design tokens. No shared logic.

PURPOSE:
Monitor 272 healthcare employers directly at the ATS source every
30-480 seconds, apply keyword matching + AI-powered resume scoring,
infer missing salaries from government and peer sources, and alert
the moment a matching Epic/EHR/Healthcare IT role appears.
Also monitors via HiringCafe wide-net scrape (1-minute cron, server-side
keyword filtering via searchState SSR) and Jobhive CSV cron scan
(1-hour, discovers roles at entirely unknown companies).

DESIGN PRINCIPLE: DO NOT ASSUME
Every displayed salary traces to a verifiable source.
Every alert traces to a job confirmed live at the ATS source.
Nothing is fabricated or guessed.

DURABLE OBJECTS — FOUR CLASSES
═══════════════════════════════════════════════════════════════

CLASS 1: PlatformDO (binding: *_DO per platform)
  One instance per ATS platform. Round-robin polls all companies on that ATS.
  Alarm intervals: greenhouse/lever/ashby 120s, workday/icims 240s,
                   successfactors/taleo 480s, oracle_hcm/infor_hcm/selectminds 480s.
  Saturday: 20min reduced polling.
  SelectMinds DO also stores 'selectminds_cursor' key for ID walk state.

CLASS 2: BatchPollerDO (binding: BATCH_POLLER_DO)
  ONE globally ('batch-poller'). Covers BATCH_WATCHLIST — general employers.
  Round-robin every 4min. 50 companies per alarm cycle, 400ms between fetches.

CLASS 3: StateStoreDO (binding: STATE_STORE)
  ONE globally ('stat-state'). Shared state store.
  Keys: recent_matches (200), seen_ids (structured — see below), company_list,
        profile (candidate JSON), unmatched_jobs (2000 cap, Browse tab),
        jobhive_scan_last, email_queue, log_buffer, seen_sweep_cursor,
        salary_* (BLS/LCA cache keys).

CLASS 4: SalaryInferenceDO (binding: SALARY_DO)
  Salary inference pipeline. Auto-refresh via cron.

SEEN_IDS — STRUCTURED FORMAT (updated S10/S11)
═══════════════════════════════════════════════════════════════

Format: Array<{id, seenAt, diedAt?, url?}> stored in StateStoreDO 'seen_ids'.
  (Was: string[] — backward compat: raw strings auto-converted to {id, seenAt:epoch0})

SEMANTICS: seen = alert dedup only. NOT Browse suppression.
  Browse captures ALL env-filtered jobs (matched + unmatched). See BROWSE below.

TTL:
  Dead entries (diedAt set) pruned after 30 days (SEEN_TTL_MS).
  Live entries pruned after 90 days (SEEN_LIVE_MAX_MS) as hard safety cap.
  Pruning on every saveSeenIds() call.
  KV.max_seen (5000) secondary cap after TTL filter.

Ghost resurrection:
  liveness 'dead' → entry.diedAt = now. Entry stays in map.
  Next encounter: if diedAt set AND liveness returns 'live' → clear diedAt, re-alert.
  Both HC cron and platform-do alarm loop implement resurrection.

Cron sweep (maybeRunSeenSweep):
  Runs every cron tick (1min) via handleScheduled().
  HEAD-requests 20 dead-with-url entries per tick.
  Cursor in StateStoreDO 'seen_sweep_cursor'. Resets to 0 at list end.
  live/3xx → delete diedAt (resurrection on next DO alarm cycle).

URL sourcing: addToSeen(seenMap, jobId, jobUrl) — url field populated from job.url.
  Sweep uses entry.url for HEAD check. Only entries with url are swept.

Platform-do local seen-set: flat string Set stored in DO storage 'seen_ids'.
  Separate from global StateStoreDO seen_ids. Local dedup only, no TTL.

BROWSE TAB — MODEL (updated S11)
═══════════════════════════════════════════════════════════════

RULE: seen ≠ discarded from Browse.
  Browse captures ALL jobs that pass passesEnvFilter() — matched AND unmatched.
  Matched jobs appear in Browse and stay discoverable after recent_matches 200-cap.
  Dedup gate (seen_ids) affects alerts only, never Browse visibility.

Before S11: Browse only captured passesEnvFilter() && !matchJob() — matched jobs
  were excluded, invisible once they aged out of recent_matches (200 entry cap).

Applied to: platform-do.js alarm loop, batch.js alarm loop, HC cron path.
  All three: `if (passesEnvFilter(job)) unmatchedJobs.push(job);`
  (variable name is legacy — "unmatched" now means "all env-filtered, non-alert path")

BOOTSTRAPDOS — SEED MERGE BEHAVIOR (updated S10/S11)
═══════════════════════════════════════════════════════════════

bootstrapDOs() merges SEED_COMPANIES into stored list on every call.
  Merge key: (ats, url) — NOT (ats, token).
  Token excluded from key because cursor-walking adapters (SelectMinds) change their
  token as the cursor advances; keying by token caused duplicate entries.
  New seed entries auto-picked up without wiping manually-added companies.

Bootstrap is idempotent:
  Platform DOs already in registry → skipped (spawned:0 is normal after first run).
  New platform (new DO class + migration) → spawned on first bootstrap after deploy.

CRON PIPELINE — THREE LAYERS
═══════════════════════════════════════════════════════════════

LAYER 1: Platform DOs (DO alarm, per-platform)
  Direct polling of 272 seed companies at their native ATS.
  Alarm interval: 120s (GH/Lever/Ashby) to 480s (SF/Taleo/Oracle/Infor/SelectMinds).

LAYER 2: HiringCafe wide-net (1-minute cron)
  searchState SSR — server-side keyword filtering confirmed 2026-06-08.
  2 calls per tick (Remote + Hybrid). v5_processed_job_data included.
  Description second-pass via /job/{requisitionId} after keyword match.

LAYER 3: Jobhive CSV scan (1-hour cron)
  Streams storage.stapply.ai CSV slices to discover Epic roles at unknown companies.
  On match: P3 email alert + maybeAddOrPromoteCompany(gate:'strict').
  Cron every minute, throttle to 1 execution/hr via jobhive_scan_last.

LAYER 4 (cron): maybeRunSeenSweep (every cron tick)
  Re-checks 20 dead seen-set entries per tick for ghost resurrection.
