WHAT STAT IS
═══════════════════════════════════════════════════════════════

STAT = "Straight To A Target"
A personal job intelligence system for Epic / Healthcare IT roles,
built on Cloudflare Workers Plus.

STAT is completely separate from FIELD (sports intelligence PWA).
No shared infrastructure. No shared design tokens. No shared logic.

PURPOSE:
Monitor 271 healthcare employers directly at the ATS source every
30-480 seconds, apply keyword matching + AI-powered resume scoring,
infer missing salaries from government and peer sources, and alert
the moment a matching Epic/EHR/Healthcare IT role appears.
Also monitors via HiringCafe wide-net scrape (1-minute cron, now
with server-side keyword filtering via searchState SSR) and Jobhive
CSV cron scan (1-hour, discovers roles at entirely unknown companies).

DESIGN PRINCIPLE: DO NOT ASSUME
Every displayed salary traces to a verifiable source.
Every alert traces to a job confirmed live at the ATS source.
Nothing is fabricated or guessed.

DURABLE OBJECTS — THREE CLASSES
═══════════════════════════════════════════════════════════════

CLASS 1: PlatformDO (binding: PLATFORM_DO)
  One instance per ATS platform (Greenhouse, Lever, Ashby, Workday,
  iCIMS, SuccessFactors, Taleo, Oracle HCM, Infor HCM). Round-robin polls
  all companies on that ATS. One alarm per platform.
  Alarm intervals: greenhouse/lever/ashby 120s, workday/icims 240s,
                   successfactors/taleo 480s, oracle_hcm/infor_hcm 480s,
                   Saturday 20min.

CLASS 2: BatchPollerDO (binding: BATCH_POLLER_DO)
  ONE globally (named 'batch-poller'). Covers BATCH_WATCHLIST —
  general employers, logistics, CS companies. Round-robin every 4min.
  50 companies per alarm cycle, 400ms between fetches.

CLASS 3: StateStoreDO (binding: STATE_STORE)
  ONE globally (named 'stat-state'). Shared state store.
  Keys: recent_matches (200), seen_ids (5000 cap), company_list,
        profile (candidate JSON), unmatched_jobs (2000 cap, Browse tab),
        jobhive_scan_last (last cron run timestamp),
        email_queue (match queue + lastSentAt for throttle),
        log_buffer (rolling alarm log entries).

CRON PIPELINE — THREE LAYERS
═══════════════════════════════════════════════════════════════

LAYER 1: Platform DOs (DO alarm, per-platform)
  Direct polling of 271 seed companies at their native ATS.
  Fastest path: alerted within one alarm cycle of posting.
  Alarm interval: 120s (GH/Lever/Ashby) to 480s (SF/Taleo/Oracle/Infor).

LAYER 2: HiringCafe wide-net (1-minute cron)
  Scrapes hiring.cafe via searchState SSR — server-side keyword filtering
  confirmed 2026-06-08 (replaces the prior ?q= no-op global feed approach).
  2 calls per cron tick (Remote + Hybrid environments).
  Returns pre-filtered ssrHits with v5_processed_job_data (salary, states, etc.).
  Description second-pass via /job/{requisitionId} after keyword match.

LAYER 3: Jobhive CSV scan (1-hour cron)
  Streams jobhive's ATS CSV slices from storage.stapply.ai to discover Epic
  roles at companies entirely outside the seed list.
  Slices: workday (50MB cap), taleo (10MB cap), icims (20MB cap).
  On match: fires P3 alert (email only) immediately. Also calls
  maybeAddOrPromoteCompany() with gate:'strict' — only health systems and
  Epic consulting firms auto-promoted to direct DO polling (after 2 matches).
  Status: quote-aware streaming CSV parser handles all edge cases.

KEY FEATURES (2026-06-07 → 2026-06-08)
═══════════════════════════════════════════════════════════════

GEO FILTER (2026-06-08 via passesEnvFilter)
  isNonUsLocation() runs before env filter and keyword matching.
  HC jobs: uses v5 workplaceStates structured data (most reliable).
  Jobhive jobs: uses countryIso CSV field.
  All others: location string analysis — conservative, only suppresses
    on clear international signals. Unknown locations always pass.
  NON_US_ISO set + NON_US_COUNTRIES list in notify.js.

EMAIL THROTTLE (2026-06-08)
  Queue-based. Matches accumulate in StateStoreDO 'email_queue'.
  Sends at most once per 4 hours (EMAIL_COOLDOWN_MS = 14,400,000ms).
  Max 6 emails/day — stays under Resend free tier (100/day).
  Per-job Pushover (P1) fires immediately; email batches across cooldown window.

URL→ATS AUTO-DETECTION (2026-06-07)
  POST /detect-ats {url} → {ats, token, url} for company adds.
  Also fires from match paths — when a jobhive/HC discovery identifies
  a company, auto-detection promotes it to the correct DO poller.

WORKER LOG BUFFER + /logs ENDPOINT (2026-06-07)
  Rolling in-memory log of alarm-cycle results stored in StateStoreDO.
  GET /logs?ats=workday — filter by ATS. Limit 200 max.
  CI log-check workflow reads and writes to outbox/ for Claude inspection.

ALERT ROUTING
═══════════════════════════════════════════════════════════════

P1 (Epic / EHR / Healthcare IT):
  Pushover HIGH priority — bypasses DND, siren sound. Per-job immediate.

P2 (Data / SQL / Analytics):
  Pushover normal priority — batched into one push.

P3 (Product / Project / IT / Jobhive Discovery):
  Email only — no push. Includes jobhive-discovered roles.

ALL: email digest (throttled 4hr, queue-based) with fit score, salary,
     cover letter opener. Fit scoring (Gemini) can downgrade P1 to
     email-only if score < threshold. Requires stored candidate profile.
     Liveness check (HEAD request) fires before every alert.
