═══════════════════════════════════════════════════════════════
WHAT STAT IS
═══════════════════════════════════════════════════════════════

STAT = "Straight To A Target"
Healthcare urgency framing. A personal job intelligence system
for Epic / Healthcare IT roles, built on Cloudflare Workers Plus.

STAT is completely separate from FIELD (sports intelligence PWA).
No shared infrastructure. No shared design tokens. No shared logic.
The two projects use the same Cloudflare account and GitHub PAT
only because they share an owner — not a codebase.

PURPOSE:
Monitor 50+ healthcare employers directly at the ATS source
every 30–120 seconds, apply keyword matching + AI-powered resume
scoring, infer missing salaries from four government and live
peer sources, and alert via Pushover + email the moment a
matching Epic/EHR/Healthcare IT role appears.

DESIGN PRINCIPLE: DO NOT ASSUME
Every displayed salary number traces to a verifiable source.
Every alert traces to a job confirmed live at the ATS source
at the moment of alert. Nothing is fabricated or guessed.

DURABLE OBJECTS — TWO CLASSES
═══════════════════════════════════════════════════════════════

CLASS 1: CompanyWatcherDO
  Binding: COMPANY_WATCHER
  Instances: one per watched employer (~50+ at bootstrap)
  DO key pattern: "{ats}:{token}" e.g. "greenhouse:nordicglobal"
  Storage per instance:
    - company: config object (name, ats, token, url)
    - seen_ids: array of job IDs seen (capped 500)
    - seen_count: integer
    - last_run: ISO timestamp
  Alarm interval: 30–120s depending on ATS platform
  Lifecycle: init via /init POST → alarm loop → reschedule forever
  Crash recovery: DO alarms survive deploys and Worker restarts

CLASS 2: SalaryInferenceDO
  Binding: SALARY_INFERENCE
  Instances: ONE globally (named 'salary-inference')
  Storage:
    - peer_pool: JSON array, rolling 14-day salary disclosures
    - peer_count: integer
    - lca_by_employer: JSON index of DOL H-1B LCA data by employer
    - lca_by_soc: JSON index of LCA data by SOC+state
    - lca_count: integer
    - bls_wages: JSON index of BLS OEWS percentiles by SOC
    - bls_fetched_at: ISO timestamp
    - lca_fetched_at: ISO timestamp
  Lifecycle: bootstrapped at first deploy, refreshed via HTTP endpoints

═══════════════════════════════════════════════════════════════
ALERT ROUTING
═══════════════════════════════════════════════════════════════

P1 (Epic / EHR / Healthcare IT):
  Pushover HIGH priority — bypasses Do Not Disturb, siren sound
  One push per job. Immediate.
  Salary prefix: "· ~$X" if inferred, "· $X" if disclosed

P2 (Data / SQL / Analytics):
  Pushover normal priority — batched into one push
  All P2 matches in current poll sent as single message

P3 (Product / Project / IT):
  Email only — no push notification

ALL priorities receive email digest when any match fires.
Email HTML includes: job title, company, location, ATS badge,
salary (disclosed/inferred with source), ghost flag, liveness flag,
cover letter opener button (see Resume Matcher).

═══════════════════════════════════════════════════════════════
GHOST JOB THRESHOLDS
═══════════════════════════════════════════════════════════════

STAT sources jobs directly from ATS APIs — confirmed live at source.
Ghost risk = posting age, not source reliability.

warn_after_days:     21  → alert includes "⚠️ Xd old — verify still open"
suppress_after_days: 90  → no alert at all

Rationale: Healthcare IT / Epic analyst roles at smaller health systems
can genuinely take 4–8 weeks to fill. These thresholds are intentionally
more lenient than the tech-industry standard (30-day suppress).

═══════════════════════════════════════════════════════════════
LIVENESS CHECK
═══════════════════════════════════════════════════════════════

Before any alert fires, STAT sends a HEAD request to the job URL
with a 4-second timeout. This confirms the URL is live at alert time —
not just at the last ATS poll (which could be 30–120s ago).

Results:
  'live'    — HTTP 200–399 → alert fires normally
  'dead'    — HTTP 4xx → suppress alert, stay in seen-set
  'unknown' — timeout or 5xx → alert fires with "⚡ URL unconfirmed" note

'dead' jobs stay in the seen-set so they don't re-trigger on the next poll.
'unknown' jobs pass through — better to false-positive than silently drop.

This liveness check is something HiringCafe explicitly does NOT do.
HiringCafe confirms jobs at crawl time (up to 12 hours ago).
STAT confirms at the moment the alert fires.

KNOWN ISSUES / PENDING WORK
═══════════════════════════════════════════════════════════════

PENDING — requires action before first deploy:
  □ Create KV namespace: wrangler kv:namespace create STAT_KV
  □ Paste KV namespace ID into wrangler.toml
  □ Set all four Worker secrets via wrangler secret put
  □ Deploy: wrangler deploy
  □ Bootstrap: POST /bootstrap

PENDING — verify before relying on:
  □ Workday tenant URLs in SEED_COMPANIES — some health systems use
    custom domains rather than *.myworkdayjobs.com. Verify against
    each employer's careers page before first deploy.
  □ Oracle Taleo HTML selectors — most fragile adapter. Taleo's HTML
    structure varies by version and tenant. If a Taleo company
    returns zero results, inspect HTML and tune selectors in adapters.js.
  □ DOL LCA XLSX parsing — binary XLSX parser in salary.js is minimal.
    If DOL files are true XLSX binary, the TSV fallback path takes over.
    Test via POST /salary-refresh and check /salary-status.

KNOWN LIMITATIONS:
  □ iCIMS large-org sitemap cap: adapters.js fetchICIMS() caps at 500
    entries. Increase if monitoring HCA/Tenet/Dignity extensively.
  □ BLS OEWS HTML parser is regex-based. If BLS changes page structure,
    parser may return null. Fallback: no salary, no crash.
  □ Taleo embedded JS blob varies by tenant. Some Taleo tenants may
    need manual selector updates in adapters.js.
