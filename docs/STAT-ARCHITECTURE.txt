FULL PIPELINE
════════════════════════════════════════════════════════════════════════════

Three polling layers run in parallel:

LAYER 1 — Platform DOs (real-time, 120-480s alarm cycles):
  PlatformDO alarm → fetchCompanyJobs(company, env) per ATS type
    → normalize jobs (adapters.js makeJob())
    → ghost filter (analyzeAge, suppress if >GHOST.suppress_after_days)
    → Browse capture (passesEnvFilter() && !matchJob() → unmatchedJobs)
    → dedup (seen_ids check — alert path only)
    → passesEnvFilter() [env + geo gate]
    → matchJob() [keyword matching against WATCH_GROUPS]
    → liveness check (HEAD request on job.url)
    → enrichJobWithSalary() [peer → LCA employer → LCA SOC → BLS]
    → companyAwarePriority() [P1/P2/P3 assignment]
    → dispatchAlerts() [Pushover P1 immediate + email queue]
    → saveRecentMatches() [rolling 200 in StateStoreDO]

LAYER 2 — HiringCafe wide-net (1-minute cron):
  runHiringCafeScrape() → fetchHiringCafe(keyword, env) × 2
    → buildHcSearchState(keyword, workplaceType) → SSR filtered hits
    → mapHiringCafeHit() → normalizedJob with job.hc enrichment struct
    → isNonUsLocation() [geo gate, HC path uses v5.workplaceStates]
    → passesEnvFilter() + matchJob()
    → fetchHcDescription(requisitionId) [second-pass, matched only]
    → liveness + salary + Maryland + dispatchAlerts + saveRecentMatches
  HC does NOT use DO alarms. Uses cron handleScheduled().

LAYER 3 — Jobhive CSV scan (1-hour cron, throttled):
  maybeRunJobhiveScan() → Range-fetch CSV slices from storage.stapply.ai
    → streaming quote-aware CSV parser (splitCSVRows + parseCSVRow)
    → countryIso gate → usLocale gate → matchesEpicTerms()
    → build job object → passesEnvFilter() → matchJob()
    → enrichJobWithSalary() → P3 dispatch (email only)
    → maybeAddOrPromoteCompany(gate:'strict') [healthcare gate for promotion]

DURABLE OBJECTS
════════════════════════════════════════════════════════════════════════════

PlatformDO (src/platform-do.js):
  One per ATS type. Single alarm, round-robin polling.
  CHUNK_SIZE = 15 companies per alarm cycle.
  Cursor persists between alarms (all companies covered in rotation).
  Exposes GET /platform/{ats} for status.
  Alarm intervals: GH/Lever/Ashby 120s, Workday/iCIMS 240s,
    SF/Taleo/Oracle/Infor 480s. Saturday: 20min (reduced polling).

BatchPollerDO (src/batch.js):
  ONE global instance ('batch-poller'). BATCH_WATCHLIST companies.
  CHUNK_SIZE = 25. Alarm every 2 minutes.
  Covers non-healthcare employers (general tech, logistics, CS).
  Exposes GET /batch-status.

StateStoreDO (src/store.js):
  ONE global instance ('stat-state'). All shared mutable state.
  Keys:
    recent_matches    Rolling 200 most recent keyword matches
    seen_ids          Dedup set (5000 cap; prevents re-alerting)
    company_list      Persisted company additions (POST /companies)
    profile           Candidate JSON from resume upload
    unmatched_jobs    2000 cap. Browse tab feed.
    email_queue       { matches: [...], lastSentAt: ts } for throttle
    log_buffer        Rolling alarm log entries for /logs endpoint
    jobhive_scan_last Last Jobhive cron run timestamp (Unix ms string)
    salary_*          BLS/LCA cache keys (managed by SalaryInferenceDO)

SalaryInferenceDO (src/salary.js):
  Salary inference pipeline: peer pool → LCA employer → LCA SOC → BLS.
  Auto-refresh via maybeRefreshSalaryCaches() in cron.
  Exposes GET /salary-status, POST /salary-refresh.

NOTIFY.JS — KEY COMPONENTS
════════════════════════════════════════════════════════════════════════════

passesEnvFilter(job):
  1. isNonUsLocation(job) — geo gate (conservative, non-US signals only)
     HC jobs: v5.workplaceStates (structured, most reliable)
     Jobhive jobs: countryIso field
     All others: location string vs NON_US_COUNTRIES list + code patterns
  2. ENVIRONMENTS filter (config.js) — 'remote' and/or 'hybrid'
  Unknown environment → let through (conservative)

matchJob(job, extraKeywords?):
  Checks title + description against WATCH_GROUPS in config.js.
  Returns { group, priority, label, matchedKw } or null.
  Extra keywords from HC v5.requirementsSummary and hc.technicalTools.

EMAIL QUEUE THROTTLE (queueAndMaybeSendEmail):
  EMAIL_COOLDOWN_MS = 14,400,000 (4 hours). Max 6 emails/day.
  Queue accumulates in StateStoreDO 'email_queue' across cron ticks.
  On cooldown elapsed + queue non-empty: drain queue, send digest, reset.
  Per-job Pushover (P1) fires immediately and independently of queue.

checkJobLiveness(job):
  HEAD request on job.url. Timeout: 4s.
  Returns: 'live' | 'dead' | 'unknown'
  'dead' (4xx): suppress alert, do NOT add to seen_ids.
  'unknown' (timeout/5xx): let through with note (false-positive better than miss).

  Salary: "Minimum Salary / Maximum Salary" text patterns.

  class="lm-richtext-content _op_PositionDescription..." from detail page.
  Hourly salary: "$N.NN - $N.NN / hour" → × 2080 for annual range.

Taleo path (legacy enrichDescriptions path — supplementary):
  enrich.js can fetch initialHistory from detail URL for description text.
  adapters.js already extracts salary + title in fetchTaleo() directly.

INDEX.JS — KEY ENDPOINTS & FEATURES
════════════════════════════════════════════════════════════════════════════

/logs (GET):
  Worker log buffer — recent alarm-cycle results stored in StateStoreDO.
  Captures per-ATS alarm results, Workday BR intercept outcomes, errors.
  Query: ?limit=N (default 50, max 200) ?ats=workday (filter by ATS).
  CI log-check workflow reads via raw.githubusercontent.com → outbox/.

/html-probe (POST):
  Generic ATS reverse engineering from Cloudflare Worker IP.
  Body: {url, maxBytes?} (default 500000 bytes).
  Returns: { visibleText (3000 chars), metaTags, jsonLd, frameworks,
             dataAutomationIds, hiddenInputs, htmlSnippet (2000 chars) }
  CF Worker IPs bypass WAFs that block GitHub runner IPs (Workday, HiringCafe,
  Oracle HCM, Infor HCM, iCIMS all confirmed accessible from CF IP).
  Also available on FIELD relay as html_probe MCP tool.

/hc-probe (GET):
  Fetch any HC URL from CF IP, return __NEXT_DATA__ analysis.
  Used for HiringCafe searchState investigation.

/detect-ats (POST):
  URL → ATS type auto-detection. {url} → {ats, token, url}.
  Pattern-matches domain against known ATS URL signatures.
  Also fires from match paths for auto-promotion of discovered companies.

/jobhive-scan (GET, debug):
  Manual trigger of Jobhive CSV scan. ?ats=workday&maxBytes=N&limit=N.

/jobhive-sample (GET, debug):
  First N bytes of a jobhive ATS CSV slice. For structure inspection.

/jobhive-manifest (GET, debug):
  Fetch jobhive manifest from storage.stapply.ai. Shows parquet URLs,
  delta slices, range support details.

/workday-probe (GET, debug):
  Rate-limit floor test for Workday SSR endpoint via Worker.

KEY ARCHITECTURAL DECISIONS
════════════════════════════════════════════════════════════════════════════

DECISION: Workday plain HTML GET (2026-06-08)
  PROBLEM: BR + DataImpulse proxy was working but expensive (~3-5s/call,
    BR quota consumption, proxy credentials as secrets).
  FINDING: Workday renders job cards server-side when ?q= is set on the
    listing URL. No JS execution needed for job discovery.
  DECISION: Replace entire BR+proxy stack with plain fetch + ?q=epic.
  RESULT: ~200-500ms/page, no quota cost, no proxy credentials.

DECISION: HiringCafe searchState SSR (2026-06-08)
  PROBLEM: ?q= and ?environment= params confirmed no-ops — global feed.
  FINDING: ?searchState= JSON param is server-side processed, returns
    filtered hits with v5 data. Confirmed via probe (52 results for
    "Epic within" Remote).
  DECISION: Replace ?q= approach with searchState-based fetching.
  RESULT: Targeted results, correct ssrTotalCount, v5 data present.

DECISION: Jobhive streaming CSV (2026-06-08)
  PROBLEM: Need to discover Epic roles at health systems not in seed list.
  SOURCE: Jobhive (stapply.ai) publishes ATS-segmented CSV files of job
    listings. Range requests allow partial reads (50MB of a 680K-row CSV).
  DECISION: Streaming O(1) memory CSV reader with quote-aware parser.
  RESULT: ~430K Workday rows processable per hour, hourly cron, P3 alerts
    on first discovery + auto-promotion to direct DO polling after 2 matches.

DECISION: All description paths → plain fetch (2026-06-08)
  PROBLEM: BR was the last remaining NEEDS_BROWSER_FETCH use (Taleo detail).
  FINDING: Taleo detail pages are server-rendered; initialHistory hidden
    input contains full job data URL-encoded. No JS needed.
  DECISION: Parse initialHistory directly in fetchTaleo(), eliminate BR
    dependency for descriptions.
  RESULT: NEEDS_BROWSER_FETCH set is now empty. BR only used for Taleo
    search page (SPA job-discovery), nothing else.

DECISION: Email queue throttle (2026-06-08)
  PROBLEM: Many matches in a burst would send too many emails, hitting
    Resend free tier limit (100/day).
  DECISION: Queue-based accumulation in StateStoreDO. Drain at most once
    per 4 hours. Per-job Pushover still fires immediately.
  RESULT: Max 6 emails/day regardless of match volume.

DECISION: Geo filter in passesEnvFilter (2026-06-08)
  PROBLEM: International jobs surfacing as "remote" (UK, CA, AU companies).
  DECISION: isNonUsLocation() gates before env filter. Conservative: only
    suppresses on clear non-US signals. Unknown locations always pass.
    HC jobs use v5.workplaceStates (most reliable). Others use location
    string vs NON_US_COUNTRIES + ISO code patterns.
